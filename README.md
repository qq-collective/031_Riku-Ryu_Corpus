# QU_TED Project

> 英語ゼミの議論を、上流から下流まで支える5つのAI。

**MASAKI × セナリ学院**｜AI協働：Claude（ヘルン）/ Gemini（シュタルク）

---

## これは何？

大学の英語ゼミで「質疑応答が活性化しない」という問題から生まれたAIアプリ群です。

英語力や専門用語の問題は「症状」であって「原因」ではない——という視点から設計しています。
本当に解くべき問いは、**どうすれば違いを楽しめる議論の場になるか**です。

5つのアプリはバンドのように連携しますが、どれか1つから始めることもできます。

## 各メンバーの紹介

### Timothy｜#026 — 発表準備・骨格設計

> *「さあ、今から5分間、君はスティーブ・ジョブズだ。——自信を持っていこう。」*

要旨やスライドを入力すると、TED風の発表骨格と「登壇者への4つの問い」を生成する。

「正しく読み上げる」から「場を動かすために話す」への転換を支援する兄貴分的な存在。Q&Aシミュレーションで、発表前の心理的安全性を高める。

→ [Timothy README](./data/026_QU_TED_Timothy/README.md)

---

### Elice｜#027 — 聴衆の声の可視化

> *「物足りないわねえ。私がちゃんとまとめるから、もっとやばい意見出しなよ。」*

発表後、聴衆の声を3カテゴリ（First Impressions / Agree Points / Another Opinions）で収集し、マトリックスに整理する。

「あなたへの質問攻め」を「みんなの地図」に変える。誰が何を感じたかが見えるから、議論が始まる。

→ [Elice README](./data/027_QU_TED_Elice/README.md)

---

### Donovan｜#028 — 議論の記録・アーカイブ

> *「今日うまく言えなくてもいい。あとから振り返ったとき、きっと意味があったってわかるから。」*

Eliceのマトリックスから議論の火種を3つ投げ、フリートークをZoom/Teams文字起こしごと受け取って記録する。

「今は弱くていい。積み重ねが資産になる」設計。後から振り返ったとき「あのとき意味のあることをしていた」と気づける記録を残す。

→ [Donovan README](./data/028_QU_TED_Donovan/README.md)

---

### Dr. Mucho｜#029 — 概念の血肉化・用語の読み合わせ

> *「これは誰が正しいかを決めるゲームじゃない。正しさの違いを楽しみに来たんだよね？」*

「この用語、なんか引っかかる」を持ち込むと、日本語・英語・例え話の3カテゴリで言い換え候補を展開する。チームで「これがわかった！」を投票して着地点を探すゲーム型ツール。

辞書を引いても解決しない。定義を覚えても腹落ちしない——そこに刺さる。

→ [Dr. Mucho README](./data/029_QU_TED_Dr.Mucho/README.md)

---

### 隼人｜#030 — 論文読解・推論スタイルの発見

> *「作曲家の意図をまず受け止めて。そのあとは、あなたの音を加えていい。——Spheresは、あなたの声も待っている。」*

論文の一節を貼り付けると、演繹・帰納・アブダクションの3つの推論方式で同時に分解する。

「英語論文が読めないのは語彙の問題ではなく、論証構造が見えていないからでは？」という問いから生まれた。どの読み方を選ぶかで、自分の思考スタイル（戦士型／僧侶型／魔法使い型）が見えてくる。

→ [隼人 README](./data/030_QU_TED_Hayato/README.md)

## 技術仕様（共通）

- 全アプリ：シングルHTMLファイル（Vanilla JS）
- AI：Anthropic Claude API（claude-sonnet）
- 外部ライブラリ：最小限（Google Fontsのみ等）
- ホスティング：GitHub Pages対応
- データ保存：なし（セッション完結型）

APIキーはAnthropicコンソール（[console.anthropic.com](https://console.anthropic.com)）で取得してください。
